# Contributing

We welcome issues and pull requests that improve clarity, correctness, and interoperability.

## Scope
Good contributions:
- clarifications to normative requirements (MUST/SHOULD)
- additional test vectors
- bug fixes to the reference verifier
- documentation improvements

Out of scope:
- adding rail integrations
- proprietary policy engines
- production control plane code

## Process
1. Open an issue describing the change.
2. Submit a PR referencing the issue.
3. Keep changes minimal and well-scoped.
4. Include test vectors when changing verification behavior.
